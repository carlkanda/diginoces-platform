# Sprint 2 Plan — Projects and Events Foundation

## 1. Sprint goal

Sprint 2 builds the project and event foundation after Sprint 1 establishes the secure platform base.

---

## 2. Sprint scope

Sprint 2 includes:

- wedding project model;
- event model;
- project code generation;
- event code generation;
- project lifecycle statuses;
- project members;
- event members;
- partner-created project draft flow foundation;
- basic project dashboard;
- workflow checklist foundation.

---

## 3. Requirements covered

Primary requirement groups:

- PROJ-001;
- PROJ-002;
- PROJ-003;
- PROJ-004;
- PROJ-006;
- PROJ-007;
- PART-003;
- PART-004;
- ROLE-001;
- ROLE-004.

---

## 4. Required deliverables

- Project database migration.
- Event database migration.
- Project member assignment model.
- Event member assignment model.
- Project code generator.
- Event code generator.
- Project lifecycle status enum.
- Basic project CRUD API.
- Basic event CRUD API.
- Permission checks for project/event access.
- Basic project list/detail UI.
- Basic event list/detail UI.
- Audit logging for project/event changes.

---

## 5. Acceptance criteria

Sprint 2 is complete when:

- admin can create a project;
- project code is generated automatically;
- admin can create events inside a project;
- event code is generated automatically;
- events belong to one project;
- unauthorized users cannot access projects/events;
- audit logs are created for project/event changes;
- basic tests pass.

---

## 6. Out of scope

Do not build yet:

- full guest management;
- RSVP;
- invitation generation;
- WhatsApp workflows;
- check-in;
- payment enforcement;
- full contract workflow.

---

## 7. Summary

Sprint 2 creates the structural base for everything else. Once project and event foundations exist, the team can build guest management, RSVP, invitations, and check-in on top of them.
