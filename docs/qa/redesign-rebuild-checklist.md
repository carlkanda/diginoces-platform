## Design Standard

The product should feel like a wedding operations atelier: a precise, calm command workspace for real event teams, not a generic SaaS dashboard and not decorative wedding stationery.

Functional design inspiration:

- Linear: dense navigation, calm hierarchy, fast scan paths.
- Stripe Dashboard: trust, permission-sensitive workflows, clear status grammar.
- Apple Human Interface: restrained surfaces, obvious controls, no ornamental complexity.
- Notion database views: readable dense records, clear empty states, direct manipulation.

Non-negotiables:

- Use shadcn/ui primitives as the structural base for pages and shared interface pieces.
- Use the best component for the job from the shadcn catalog, not custom styled blocks by default.
- Keep every page purpose-led: title, primary user question, next action, and safe navigation must be clear.
- Keep Diginoces identity distinct: deep teal, measured gold, clean neutral canvas, logo presence, event-operations language.
- Do not use generic hero cards, repeated metric-card grids, decorative gradients, glassmorphism, beige wedding-paper defaults, or sprint/build wording.
- Do not change product behavior while redesigning.
