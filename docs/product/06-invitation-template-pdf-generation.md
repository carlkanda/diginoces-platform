# 06 — Invitation Template & PDF Generation

## 1. Purpose

This document defines how the Diginoces platform should handle Canva-exported invitation templates, dynamic field placement, QR code generation, invitation PDF generation, preview, validation, regeneration, and file/version tracking.

---

## 2. Core decision

Canva remains the creative design tool. The Diginoces web app does not replace Canva for design.

The app imports Canva-exported PDF templates and performs operational personalization:

- guest title;
- guest name;
- full guest display name;
- table name;
- table number/table ID;
- QR code;
- couple name;
- wedding date;
- wedding type;
- venue;
- invitation ID;
- RSVP link;
- wishes link;
- custom fields.

---

## 3. Template workflow

1. Diginoces designs invitation in Canva.
2. Couple approves design manually outside the app.
3. Diginoces/admin records manual approval in the app.
4. Canva PDF is uploaded as event invitation template.
5. Diginoces/admin configures dynamic fields in the visual editor.
6. System generates sample previews.
7. Diginoces/admin approves technical preview.
8. Invitations are generated by event, table, selected guests, or all.

---

## 4. Visual drag-and-drop editor

The app must include a visual editor allowing staff to:

- upload PDF template;
- select page;
- drag dynamic fields onto the page;
- resize text/QR areas;
- choose font, size, color, alignment;
- preview with sample guests;
- save template configuration;
- regenerate previews.

This replaces manual JSON coordinates.

---

## 5. PDF generation rules

The generator must support:

- personalized guest names;
- smart text fitting;
- automatic font-size reduction;
- balanced line splitting for long names;
- custom fonts;
- QR insertion;
- compression/optimization for WhatsApp sharing;
- event-specific invitation IDs;
- version history.

Generation should run as a background job, not a blocking UI request.

---

## 6. QR code rules

The system must maintain two separate QR/link concepts:

| QR/link | Purpose |
|---|---|
| Public guest page QR/link | RSVP, invitation download, wishes, event details |
| Check-in QR/link | Staff-only wedding-day check-in |

QR tokens must be secure, short, unique, revocable, and event-specific when used for check-in.

---

## 7. Preview/staging mode

Full generation must be blocked until Diginoces/admin approves a technical preview.

Preview samples should include:

- short name;
- long name;
- Couple title;
- special title;
- guest with table assignment;
- guest with QR code.

---

## 8. Validation before generation

The system must block invitation generation when required data is missing.

Required fields may include:

- title/type;
- guest name;
- event assignment;
- WhatsApp number for digital guests;
- table assignment when needed;
- public guest token;
- check-in token;
- invitation template.

Printed-only guests may bypass missing WhatsApp number with Diginoces/admin override.

---

## 9. Generation modes

The system should support:

- preview generation;
- generation by event;
- generation by selected guests;
- generation by table;
- generation by status;
- generate all;
- regenerate selected guests only.

Batch-first generation is recommended.

---

## 10. Statuses and versioning

Invitation records should track:

- not generated;
- preview generated;
- generated;
- needs regeneration;
- sent;
- resent;
- failed;
- cancelled;
- printed-only.

Regenerated invitations must preserve older versions internally, while guests download only the latest active version.

---

## 11. Acceptance criteria

- Canva PDF can be uploaded.
- Dynamic fields can be placed visually.
- Preview samples can be generated.
- Mass generation is blocked until technical approval.
- Invalid guests are blocked from generation.
- Generated PDFs include dynamic text and QR code.
- Generated files are stored in app-owned storage.
- Regeneration updates active file version.

---

## 12. Summary

The invitation module must transform Canva designs into controlled, data-driven, personalized invitations. It should preserve Canva as the design tool while making the Diginoces app responsible for validation, PDF generation, QR creation, tracking, regeneration, and file access.
